# P1-S2 — Operator Generates Admin Secret in Prod Mode [ACCEPTANCE]

Who: The operator.

What: The operator runs aactl init --mode=prod for production deployment.
Only the bcrypt hash is stored on disk. The plaintext is shown once and
never saved — the HashiCorp Vault pattern.

Why: If the plaintext were stored on disk in production, any process that
reads the config file gets the admin secret. Bcrypt-only storage means
even a compromised config file doesn't reveal the secret.

How to run: Clean temp dir, run aactl init --mode=prod, verify bcrypt hash in file.

Expected: Config has MODE=production, ADMIN_SECRET starts with $2a$12$, plaintext NOT in file.
